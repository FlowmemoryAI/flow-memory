# Mission Control UX Audit

Status: public-alpha dashboard polish audit for the local Flow Memory Mission Control experience.

## Current routes

- `dashboard/src/app/mission-control/page.tsx` — primary Mission Control route using replay data.
- `dashboard/src/app/agents/page.tsx` — agents screen scaffold.
- `dashboard/src/app/economy/page.tsx` — economy screen scaffold.
- `dashboard/src/app/neural/page.tsx` — neural screen scaffold.
- `dashboard/src/app/rl-arena/page.tsx` — RL Arena screen scaffold.
- `dashboard/src/app/audit/page.tsx` — audit screen scaffold.

## Current components

- `MissionControlCanvas` renders the main network surface.
- `AgentNode3D`, `TaskPulse`, `MemoryFlow`, `EconomyEdge`, `NeuralHalo`, `PredictionArc`, and `SafetyGate3D` map telemetry into visual primitives.
- `NetworkLegend` explains color/shape semantics.
- `AgentDetailDrawer` provides selected-agent context.
- Panels exist for runtime, agents, neural, economy, RL, and audit state.

## Data modes

- Mock mode: supported as a clearly labeled fallback through dashboard mock data helpers.
- Replay mode: supported through `dashboard/src/mock-data/local-network-replay.json`, generated by `scripts/export_visual_replay.py`.
- Live local API mode: supported structurally through `dashboard/src/lib/api.ts` and local endpoints `/visual/state`, `/visual/events`, and `/network/run-scenario`.

## Visual gaps before this polish pass

- Mission Control surface was readable but too sparse for demo screenshots.
- Agent roles did not have enough shape/position/semantic distinction.
- Economy, neural, memory, safety, and RL visuals were text-first rather than visually encoded.
- Replay controls were missing.
- Panels showed counts instead of actionable telemetry.
- Live/local API UX did not clearly show offline/error states or mode semantics.
- Demo replay did not expose enough lifecycle granularity for bid, assignment, escrow, verification, settlement, dispute, and slashing storytelling.

## Priority polish tasks

1. Make the canvas feel like a controlled scientific network view, not a landing-page background.
2. Encode visual semantics directly from telemetry fields: role, reputation, risk, neural score, economy kind/status, RL reward, safety approval, and provenance.
3. Add replay controls and event filters for demos and screen recordings.
4. Upgrade panels to show current state, recent events, and source event IDs.
5. Improve mock/replay/live mode copy and disconnected local API behavior.
6. Regenerate replay data from real local network scenarios and validate it.

## Claim control

Mission Control remains a public-alpha local/replay/API dashboard scaffold. It is connected to local Flow Memory telemetry, but it is not a hosted production console and does not imply production security, audited contracts, mainnet readiness, or hardened sandboxing.
